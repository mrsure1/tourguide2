# 🚀 웹사이트/웹앱 런칭 전 최종 체크리스트

서비스 런칭 전, 사용자 경험(UX), 보안, 성능, 그리고 운영의 안정성을 보장하기 위해 반드시 확인해야 할 항목들입니다.

---

## 1. 기능 및 사용자 경험 (Functional & UX)
- [ ] **모든 링크 확인**: 내부 페이지 링크 및 외부 링크(SNS, 파트너사 등)가 깨지지 않고 정상적으로 연결되는지 확인.
- [ ] **폼(Form) 유효성 검사**: 회원가입, 로그인, 문의하기 등의 모든 폼에서 필수 입력값 체크 및 에러 메시지가 정상 출력되는지 확인.
- [ ] **결제 프로세스 (Toss, PayPal)**: 실제 결제가 성공적으로 이루어지는지(샌드박스 및 실결제 테스트), 결제 실패 시 처리가 적절한지 확인.
- [ ] **에러 페이지 (404/500)**: 존재하지 않는 페이지 접근 시 404 페이지가 커스텀 디자인으로 표시되는지 확인. 서버 에러 시 사용자 친화적인 안내 메시지 제공 여부.
- [ ] **동작 일관성**: 브라우저 '뒤로 가기' 버튼이나 새로고침 시 상태 유지가 적절한지 확인 (Zustand 스토어 등).

## 2. 디자인 및 반응형 (Design & Responsive)
- [ ] **반응형 테스트**: iPhone, 안드로이드, 태블릿, 다양한 데스크탑 해상도에서 레이아웃이 깨지지 않는지 확인.
- [ ] **브라우저 호환성**: Chrome, Safari, Edge, Firefox 등 주요 브라우저에서 동일하게 작동하는지 확인.
- [ ] **파비콘(Favicon)**: 브라우저 탭에 표시되는 아이콘 및 앱 아이콘 설정 확인.
- [ ] **폰트 가독성**: 모든 폰트가 정상적으로 로드되며, 가독성에 문제가 없는지 확인.
- [ ] **다크 모드**: 다크 모드 지원 시 테마 전환이 매끄럽고 텍스트 가시성이 확보되는지 확인.

## 3. SEO 및 마케팅 (SEO & Marketing)
- [ ] **Meta 태그**: 각 페이지별로 적절한 `title`과 `description`이 설정되어 있는지 확인.
- [ ] **Open Graph (OG)**: 카카오톡, 페이스북, 슬랙 등에 공유했을 때 미리보기 이미지와 텍스트가 올바르게 나오는지 확인.
- [ ] **사이트맵(sitemap.xml)**: 검색 엔진 인덱싱을 위한 사이트맵 생성 및 제출 준비.
- [ ] **robots.txt**: 검색 엔진 로봇의 접근 허용/차단 설정 확인.
- [ ] **분석 도구 (GA4/Pixel)**: Google Analytics 등 방문자 추적 스크립트가 정상 삽입되었는지 확인.

## 4. 성능 및 최적화 (Performance)
- [ ] **이미지 최적화**: `next/image`를 사용하여 WebP 형식 제공 및 적절한 크기로 리사이징되는지 확인.
- [ ] **Lighthouse 점수**: 성능(Performance), 웹 표준(Accessibility), SEO 등의 점수가 양호한지(최소 80점 이상 권장) 측정.
- [ ] **빌드 결과물 확인**: `npm run build` 시 에러나 경고가 없는지 확인.
- [ ] **캐싱 전략**: 정적 페이지(Static Generation)와 동적 페이지(Server Rendering)의 캐싱 설정이 적절한지 확인.

## 5. 보안 및 데이터 (Security & Data)
- [ ] **환경 변수 (.env)**: API 키, 데이터베이스 비밀번호 등 민감 정보가 소스 코드에 노출되지 않고 `.env` 파일로 관리되는지 확인.
- [ ] **Supabase RLS**: 데이터베이스 테이블에 Row Level Security 설정이 되어 있어 외부에서 무단 접근이 불가능한지 확인.
- [ ] **SSL (HTTPS)**: 모든 통신이 HTTPS로 보호되는지 확인 (Cloudflare/Vercel 기본 제공 확인).
- [ ] **API 할당량**: Gemini API 등 외부 라이브러리의 무료 티어 할당량이 실제 운영 시 충분한지, 초과 시 처리가 되어 있는지 확인.

## 6. 인프라 및 운영 (Infrastructure)
- [ ] **도메인 연결**: 구매한 도메인이 운영 서버에 정상 연결되었는지 확인.
- [ ] **이메일 발송**: 회원가입 인증, 비밀번호 재설정 이메일 등이 실제 수신되는지 확인.
- [ ] **에러 모니터링**: Sentry 등 런타임 에러를 실시간으로 추적할 수 있는 도구 설정 여부.
- [ ] **프로덕션 DB**: 개발용 데이터(더미 데이터)를 정리하고 운영 서버 전용 데이터베이스가 준비되었는지 확인.

## 7. 가이드 및 법적 검토 (Content & Legal)
- [ ] **약관 및 정책**: 서비스 이용약관, 개인정보 처리방침 페이지가 현행 법규에 맞게 작성되었는지 확인.
- [ ] **더미 텍스트 제거**: 'Lorem Ipsum' 등 개발 중에 사용한 가짜 텍스트나 테스트용 이미지가 남아있는지 전수 조사.
- [ ] **오탈자 검수**: 주요 문구 및 버튼 텍스트의 오탈자 확인.
- [ ] **고객 지원**: 사용자 문의를 받을 수 있는 이메일이나 채널톡 등이 정상 작동하는지 확인.

---

> [!TIP]
> 런칭 직후에는 예상치 못한 버그가 발생할 수 있습니다. 초기 사용자 피드백을 빠르게 수집할 수 있는 창구를 마련해 두는 것이 좋습니다.
